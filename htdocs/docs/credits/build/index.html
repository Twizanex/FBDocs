Credits Tutorial

Advanced Topics › Credits › Credits Tutorial

Facebook Credits is a payment system that offers a safe, easy and fast way to pay for digital and virtual goods in games and apps across Facebook. This document walks through the set of simple steps you can follow to integrate with Facebook Credits on your app.

Making your app Credits Enabled

1. Designate your company's country of operation.

To enable your app for Credits, please go to the App Dashboard and click "Edit Settings" for the app you wish to enable. Then, click on "Credits" on the left-hand rail to view the credits information for your app. First, if you haven't already done so, you will want to select your company's country from the drop-down, then save those settings.

2. Register your Company and Payout Method.

To do this, click the 'register your company' link which will take you to our developer payout application. Fill out the required company and payout information before saving and proceeding, note you may not readily change this information so please ensure it's accuracy.

We highly recommend that you fill out your Company and Payment information so that Facebook can pay you out. If you fail to do so, then even though users are spending in your application you will not receive any payment from Facebook.

Finally fill out the payment information and accept our terms.

Note: As we continue to grow our payments system we are continually expanding to new countries around the world. However, at this time we are only able to payout to a select set of countries. Please see this section in our reporting and payout docs.

3. Associate Your Newly Registered Company to Your App

You will now need to associate your company to your app in order to be paid out. You can select this from the dropdown menu after checking the 'add your payout information' box, then selecting your company name. If you are just using this app for testing, you may skipp associating your company to your app and just list your test users uids under the test users section. If you plan to test making actual transactions, you must associate your company to your application and remove yourself from the test user section.

4. Set up the Credits callback

You also need to define a callback URL for your application which is necessary or the payments API to work as expected. Facebook will call this URL after a user has clicked on the Facebook Credits button in your application. This Callback needs to handle various communications between Facebook and your application. The document "Credits Callback" provides detailed information on what logic needs to exist here.

Note: This can be changed by the developer at will, unlike the company information.

5. Manage Test settings

You can add your Facebook ID as well as ids of other users who you wish to declare as testers for your application. Orders from users declared here are not processed and you will never be charged for any transaction in your app.

Please Note: Since test users are never charged for the purchase, they do not generate actual Order objects and thus will not show up in your daily reports or through our Order API. This will allow you to test that your pay dialog and callback are properly setup - but you will need to generate actual purchases in your game to fully test the reporting mechanisms we offer, which can then easily be refunded.

Now you can Save the settings. Congratulations, your app is now fully Credits enabled! Once your front-end pay dialog as well as your callback are properly setup, you can immediately start accepting payments in your app with update order.

Integrating credits into your app

Once your application has been credits-enabled, you can begin setting up your test environment by reviewing the pay dialog and callback documentation. Links and more information on our Facebook Credits branding assets can be found here. You will need to setup your front-end code which renders the pay dialog in addition to your callback which is used to communicate necessary order information with Facebook

Web Sites integrating Facebook (Connect), Native iOS Apps and Mobile Web Apps running in the Facebook iOS App can not use Facebook Credits.

Credits as a Payment

Using the pay method, you can open a modal pay dialog that will contain specific information about the item. The modal dialog will contain an item's title, description, its price in Credits, and an image of the item.

Note: Do not construct your item information (order_info) in your front-end code, as it's susceptible to user manipulation. It's best practice to pass in internal reference, usually a string, in the order_info field which is then used by your callback to determine the proper item, and return the array of parameters.

Below is a code snippet that shows how to call the Facebook Credits API in JavaScript for both canvas iframe applications and mobile web apps. To test this quickly, you should use the Credits Sample App on Github.

Pay dialog Attributes for Credits as a Payment:

Name Description
method set to pay
order_info your internal key to associated product information
action set to buy_item
dev_purchase_params JSON Object {'oscif': true}

Making call using the JS SDK: 

// This example requires callback.php to be enabled and coded.

 <html>
    <head>
      <title>My Facebook Credits Page</title>
    </head>
    <body>
      <div id="fb-root"></div>
      <script src="http://connect.facebook.net/en_US/all.js"></script>
<p> <a onclick="placeOrder(); return false;">Buy Stuff</a></p>

<script> 
    FB.init({appId: <your_app_id>, status: true, cookie: true});

    function placeOrder() {
      // If using mobile, this is where you place the
      // iOS native app check for credits (see below)

      // Assign an internal ID that points to a database record
      var order_info = 'abc123';

      // calling the API ...
      var obj = {
        method: 'pay',
        order_info: order_info,
        action: 'buy_item',
        dev_purchase_params: {'oscif': true}
      };

      FB.ui(obj, callback);
    }
    
    var callback = function(data) {
      if (data['order_id']) {
        return true;
      } else {
        //handle errors here
        return false;
      }
    };

    function writeback(str) {
      document.getElementById('output').innerHTML=str;
    }
</script>
</body>
 </html>

The javascript callback also contains order data on success, or an error message when the order is cancelled.

Here is a sample JSON payload provided to the javascript callback upon successful payment:

    {
       "order_id":9006133015289,
       "status":"settled"
    },
       "hiding":1
    },
       "onload":[
       "GiftCredits.setPurchaseLock(false)"
    ],
       "lid":"5673516601143979556"
    }

Here is a sample JSON payload provided to the javascript callback upon a user canceled payment:

    {
       "error_code":"1383010",
       "error_message":"User canceled the order."
    }

We refer to credits as a Payment on Facebook as 'oscif' which 'Optimized sole cash-in flow'.

If you need to open the pay dialog in a new page, please refer to the pay dialog documentation.

Get More Credits

This feature is an API call that allows users to purchase credits without having to purchase an item with that purchase.

Pay Dialog Attributes for Getting More credits:

    method string - new JS SDK method to initiate credits purchase dialog

Making call using the JS SDK: 

 <html>
    <head>
      <title>My Facebook Credits Page</title>
    </head>
    <body>
      <div id="fb-root"></div>
      <script src="http://connect.facebook.net/en_US/all.js"></script>
<p> <a onclick="getMore(); return false;">Get More</a></p>

<script>
 function getMore(){
    // Initialization
    FB.init({appId: <your_app_id>, status: true, cookie: true});

    // calling the API ...
    var obj = {
         method: 'pay',
         action: 'buy_credits'
    };

    FB.ui(obj, callback);
 }
</script>
</body>
 </html>

Credits on Mobile

The pay dialog can also be rendered in a mobile environment in the same way as a canvas app. For more information on mobile platform, please see our mobile documentation. Facebook Credits are not supported within iOS native apps. To detect whether your app is running within a native app, please place the following code in your app to properly suppress credits on iOS: 

if (FB.UA.nativeApp()) {
  // add code to hide your Credits flows

Get Order information via Graph API

You can interact with orders created by the application using the Facebook Credits Graph API to view and update orders as needed. More information about the specifics of the API is available in the Graph API order object document.

Note: You must use actual orders when calling the Graph API. Orders from test users will not be processed.

GET https://graph.facebook.com/[order id]?access_token=APP_ACCESS_TOKEN

Returns: a JSON object:

            {
               "id": "",
               "from": {
                  "name": "",
                  "id": ""
               },
               "to": {
                  "name": "",
                  "id": ""
               },
               "amount": ,
               "status": "",
               "application": {
                  "name": "",
                  "id": ""
               },
               "country": "",
               "created_time": "",
               "updated_time": ""
            }

You can also updates the status and other fields of an existing order using the Graph API order object. This API is throttled to 100 calls per minute.

Get Orders for your user or app

You can access orders for the given application or user by issuing an HTTP GET request to the APP_ID/payments or USER_ID/payments connection with an app access_token. The USER_ID must have added your application.

Note: currently we will only return 100,000 orders for a single date/time range. If you need to pull more than 100,000 orders for a single 24 hour period, then please split the day into multiple dayparts and make multiple queries.

API 

GET https://graph.facebook.com/[app id]/payments?status=STATUS&since=SINCE&until=UNTIL&access_token=ACCESS_TOKEN
GET https://graph.facebook.com/[user id]/payments?status=STATUS&since=SINCE&until=UNTIL&access_token=ACCESS_TOKEN


Commonly used filters

You can filter the set of orders returned by this API by to list only the orders with a particular status or orders which were placed within a particular time interval.

Name Description
status Return orders with the specified status. Specify one of "settled", "refunded", or "disputed".
since Unix time
until Unix time. You cannot exceed 24 hours between your start and end times. If you need to pull a full week, simply call your getOrders 7 times, each time for a full 24 hour day
