Feed Gaming

Getting Started › Apps on Facebook › Feed Gaming

Developers can leverage in-feed gaming to create engaging stories for users. This involves publishing a story on behalf of a user which has a flash object attached. When a user's friends see these attachments, they will be able to click to play them inline. This opens up some interesting scenarios for a small embeddable version of your game to be shown to a user's friends, at the end of which you can encourage them to click through to your canvas app.

Example use cases:

  • Post a high score on a particular level and challenge friends to beat it.
  • Enable a replay of interesting user activity so friends can see how to complete a level for example.
  • Create a mini-game that users can play in-line to get them excited to play the game.

This can be a great way to give new users a quick preview of your gameplay, and allow competition and sharing between existing players.

Example posts from real games:

  • Angry Birds lets users challenge friends to beat your score on a level. Try now.
  • Bubble Witch Saga enables sharing coins with friends, they can earn a multiplier for a high score.Try now
  • Idle Worship lets users share a mini-game which gives friends a glimpse of the full game.Try now.
  • Tetris Battle allows publishing a replay of a two-player battle. Try now.

Note: The user who is interacting with the embedded game will be anonymous. No signed_request will be passed, and it is not possible to initialize the JavaScript SDK. It is not possible to determine whether this user has previously installed your game or not, as such you will need to consider how to make the embedded experience useful for both new and existing players.

Publishing a Feed Gaming story

To publish a feed gaming story you need to

  1. Create an Open Graph object for the attachment.
  2. Publish the story on behalf of the user.
  3. Track story engagement

1. Create an Open Graph object for the attachment

First create an Open Graph object which implements the og:video meta tag. This object will be used to generate the attachment when you publish a story on behalf of a user.

Name Description Required
fb:app_id App ID that owns this object. true
og:type This can be any valid open graph type, for example game. true
og:url The unique URL of the game story. true
og:title Title of the story being shared, e.g. 'Beat Gareth's high score'. true
og:description The description for this game story. true
og:image An image URL which represents your object within the graph. The image must be at least 50px by 50px and have a maximum aspect ratio of 3:1. We support PNG, JPEG and GIF formats. true
og:video The URL of the SWF file to be embedded. true
og:video:type Determines how to render this swf - should be set to 'application/x-shockwave-flash'. true
og:video:width The width in pixels of this video when embedded. true
og:video:height The height in pixels of this video when embedded. true 
 <head prefix="og: http://ogp.me/ns fb: http://ogp.me/ns/fb">
   <meta property="fb:app_id"       content="APP_ID" />
   <meta property="og:type"         content="game" />
   <meta property="og:url"          content="http://example.com/embed/" />
   <meta property="og:title"        content="Beat Gareth's high score" />
   <meta property="og:description"  content="Gareth scored 2800, can you beat it?" />
   <meta property="og:image"        content="http://example.com/game.png" />
   <meta property="og:video"        content="http://example.com/game.swf" />
   <meta property="og:video:width"  content="398" />
   <meta property="og:video:height" content="299" />
   <meta property="og:video:type"   content="application/x-shockwave-flash" />

See Attaching Audio and Video Data for more information.

2. Publish the story on behalf of a user

Think about an appropriate place in your game to publish on behalf of a user, for example after the user completes a level.

Feed Dialog example

The recommended way to publish is using the Feed Dialog, which requires no extended permissions. Specify the url of your open graph object in the link property.

 function postToFeed() {

   // calling the API ...
   var obj = {
     method: 'feed',
     link: 'http://example.com/embed/'
   };

   function callback(response) {
     console.log(response);
   }

   FB.ui(obj, callback);
 }

See the Feed Dialog reference documentation for a complete example.

Graph API example

It is also possible to publish using the Graph API, by issuing an HTTP POST to the /USER_ID/feed connection on the User object. You will need the publish_actions permission. See Publishing Permissions for more information.

 POST /USER_ID/feed?
   link=http://example.com/embed/
   access_token=USER_ACCESS_TOKEN

When your story is rendered on Facebook it will appear as a link, with a 90x90 pixel (max) image that has a play icon overlay. The title and description, plus any user message, will also be displayed:

When a user clicks the play icon, the content will be replaced with the SWF that was specified in your object's og:video tag. There are size constraints depending on where the story is rendered on Facebook - if you want to avoid proportional scaling, it is recommended that you do not exceed 398 pixels in either dimension.

3. Track story engagement

When publishing on behalf of a user, you can specify a value for the ref parameter which is used to define a category in Facebook Insights which enables you to measure the performance of different types of posts. When viewing Insights for your app, navigate to Traffic > Stream Publish, and you will see a Story Type drop down menu at the top of the page, allowing you to filter the data for stories published, impressions and clicks by each category. This makes it easy to test different copy, images and integration points to see which performs best.

This value will also be passed back to you in the fb_ref parameter which will be appended onto the end of the url when users click on your story title and are taken directly to the game - i.e. without playing the embedded content.
