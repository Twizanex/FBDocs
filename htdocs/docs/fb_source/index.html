fb_source Parameter

Getting Started › Apps on Facebook › fb_source Parameter

This document lists the complete list of values for the fb_source parameter that is returned with the url when the user visits your app from some location on www.facebook.com like bookmarks, feed or timeline. Developers can use this parameter to better understand where their traffic is coming from so that they can better optimize their Facebook integration.

We also surface this data in your app's insights. You can find this data by navigating to the 'Users' section of insights as shown below.

If you scroll to the bottom of the 'Users' section of insights you will see a subsection under 'Referrals' which contains a list of the fb_source parameters designating your sources of traffic on Facebook.com during the designated time period.

List of Parameter Values

Canvas bookmarks

This list refers to the set of fb_source parameters returned when a user clicks on the bookmarks on the canvas page.

Name Description of Where Surfaced More Info
canvasbookmark The canvasbookmark reference can be found when your app is visited from a bookmark located on a canvas page (i.e. from apps.facebook.com/*). More Info
canvasbookmark_more The canvasbookmark_more reference can be found when a user hovers over the card icon bookmark on a canvas page, and selects your app from the dropdown that is rendered. More Info
canvasbookmark_recommended The canvasbookmark_recommended reference can be found when a user is brought to your app via the recomendations section More Info

Homepage bookmarks

This list refers to the set of fb_source parameters returned when a user clicks on the bookmarks on the user's homepage.

Name Description of Where Surfaced More Info
bookmark_apps This reference can be found when your app is visited from a bookmark on the user's homepage under the 'apps' section. bookmark_favorites This reference can be found when your app is visited from a bookmark on the user's homepage under the 'favorites' section. More Info
bookmark_seeall This reference can be found when the user clicks the 'more' link found in the 'apps' section of their bookmarks (which takes the user to the bookamrk details page), and then then visits your app from a bookmark found on that page. More Info

This list refers to the set of fb_source parameters returned when a user clicks on the bookmarks on the App Dashboard.

Name Description of Where Surfaced More Info
dashboard_toplist The dashboard_toplist value can be found when a user is brought to your app through the list of top apps in the app dashboard. More Info
request When a user clicks on a request, we set the fb_source value to request. More Info

News feed

This list refers to the set of fb_source parameters returned when a user clicks on a News feed story on the Homepage.

Name Description of Where Surfaced More Info
feed_highscore The feed_highscore value can be found in a highscore story for games that is rendered in the newsfeed. More Info
feed_music The feed_music value of fb_source designates that the story clicked by the user was a music related story. More Info
feed_opengraph The feed_opengraph value of fb_source covers all other open graph instances beyond the built-in actions such as scored, achieved, read, watched, listened and will be included in your stories when publishing custom actions. More Info
feed_passing If your app is using the scores API, we can generate passing stories when one of your users passes their friend. This feed_passing value for fb_source designates traffic coming these stories. More Info
feed_playing Facebook will automatically generate 'is playing' stories for a user who visits your app when social discovery is enabled and your app is categorized as a game. These stories contain the feed_playing fb_source to designate this traffic. More Info
feed_video The feed_video value can be found from any video story rendered in newsfeed. More Info

Ticker

This list refers to the set of fb_source parameters returned when a user clicks on a story in the Ticker.

Name Description of Where Surfaced More Info
ticker If a user is brought to your app by any ticker story, we add the fb_source value of ticker to the URL. More Info

Timeline

This list refers to the set of fb_source parameters returned when a user clicks on a story in the user's Timeline.

Name Description of Where Surfaced More Info
timeline_og When a user is brought to your app via a timeline Open Graph story, we add the timeline_og value to fb_source. More Info
timeline_news When a user is brought to your app via a news timeline story, we add the timeline_news value to fb_source. More Info
timeline_passing When a user is brought to your app via a scores passing timeline story, we add the timeline_passing value to fb_source. More Info
timeline_recent When a user is brought to your app via a link in the recent activity section on timeline, then we add the timeline_recent value to fb_source. More Info

other_multiline The can be found when a user is brought to your app through an open graph multi-line story. Currently this fb_source param can be found in stories on timeline, ticker and newsfeed. More Info
