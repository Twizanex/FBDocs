Creative Specs

Advanced Topics › Ads API › Creative Specs

Creative specs are used to define what the creative of an Ad group looks like. Creative Specs are used in Ad Creative and Ad group objects. Creative Specs can also be used to generate previews of what a creative will look like when used in an Ad group

Fields

The following are the fields for the creative spec. The required fields are different based on the creative type, see the parameter columns of the "Ad Creative Types" table below.

Name Description Permissions Returns

object_id The Facebook object ID that is relevant to the ad and ad type. See connection objects) ads_management int
body The body of the ad, not applicable to Sponsored stories ads_management string
image_hash Image ID for an image you can use in creatives and thus in ads, see Using Images ads_management string
image_url A URL for the image for this creative, see Using Images ads_management string

url_tag A string which will be appended to urls clicked from type 25 and 27 creatives. ads_management string
preview_url The URL to preview the ad, only for the current session user ads_management string

action_spec A JSON string defining an action performed by a user for use in a type 25 ad. See action spec reference for more details. ads_management JSON String

Limits

The following are the limits on the fields in a Creative Spec.

  • The maximum amount of characters that can be in an ad’s title : 25 characters
  • The maximum amount of characters that can be in an ad’s body : 90 characters
  • The maximum amount of characters that can be in a URL : 1024 characters
  • The maximum allowable length of an individual word in an ad’s text or title : 20 characters
  • The minimum amount of characters that can be in an ad’s title : 1 character
  • The minimum amount of characters that can be in an ad’s body :1 character

Action Specs for Sponsored Stories

You can create Sponsored Stories using Action Spec framework with type 25 Ad Creatives.

All legacy Ad Creative Types for Sponsored Stories can be defined using Action Spec. The following table lists such legacy types and their associated Action Spec definitions. However, note that Action Spec framework enables you to create many other Sponsored Stories beyond the legacy types below. To see other examples, go here.

Legacy Creative type Description Equivalent Action Spec
8 Promotes stories published through an app. {'action.type':'post', 'application':APP_ID}
9 Promotes stories about a user liking a page/place. {'action.type':'like', 'page': PAGE_ID}
10 Promoting stories about a user checking into a place. {'action.type':'checkin','page':PAGE_ID}
10 Promoting stories about a user checking into any child place of this page. {'action.type':'checkin', 'page.parent':PAGE_ID}
16 Promotes stories about a user using an app note. {'action.type':'app_use', 'application':APP_ID}
17 Promotes stories about a user liking any post from a page that was published by page's admin. {'action.type':'like', 'post.author':PAGE_ID,'post.wall':PAGE_ID}
19 Promotes stories about a user liking a domain. {'action.type':'like', 'object.domain':DOMAIN_ID}
19 Promotes stories about a user sharing something from a domain. {'action.type':'post', 'object.domain':DOMAIN_ID}
25 + query_template=6 Promotes stories about a user liking a specific page post. {'action.type':'like', 'post':PAGE_POST_ID}
25 + query_template=7 Promotes stories about a user commenting on a specific page post. {'action.type':'comment','post':PAGE_POST_ID}
25 + query_template=7 Promotes stories about a user commenting on any post from a page that was published by page's admin. {'action.type':'comment', 'post.author':PAGE_ID,'post.wall':PAGE_ID}
25 + query_template=8 Promotes stories about a user RSVPing into an event. {'action.type':'rsvp', 'event': EVENT_ID}

Please note that query_template is being retired from the API and is not used in the Action Spec framework.

Note: Previously, a person's app used Sponsored story is eligible to be shown to his/her friends if he/she used the app longer than 10 minutes in the last 28 days or used the app twice within the last 28 days. We made a slight change to show these Sponsored stories if a user used the app for at least 4 minutes in the last 14 days or used the app twice ever with one of those times occurring in the last 14 days.

Open Graph sponsored stories for timeline apps

In addition to on Facebook Activity sponsored stories, Custom Open Graph activity, which is generated by a timeline apps, can also be used to generate Sponsored Stories. Open Graph sponsored stories are defined in the same way as on Facebook Activity Sponsored Stories, i.e. as type 25 creative type with an action spec to define the activity to boost into a sponsored story.

Tracking Parameters

For ads that lead to content off Facebook, it may be useful to distinguish whether clicks coming to your Facebook application of website are from an ad or a news feed story. Several ad types support adding query parameter to ads destination URL for purpose of tracking.

  • For type 1 ads, you can simply append a query parameter to a website URL when specifying url_link.
  • For type 2 ads that land on an iframe page tab application, you can append a special query parameter app_data to the page tab URL when specifying url_link (see Obtaining Objects for obtaining page tab URLs). You can retrive app_id from signed_request parameter that Facebook platform appends to the application's tab URL when the tab is loaded (see Signed Request).
  • For type 4 ads, you can simply append a query parameter to n app URL when specifying url_link.
  • For type 25 ads you can use url_tags to specify a list of query parameters for the following action spec formats: {'action.type':'post', 'application':APP_ID}, {'action.type':'app_use', 'application':APP_ID} and {'action.type':'post', 'post.domain':DOMAIN_ID}. These query parameters will be appended to the ad's URL, which points to an external website or a Facebook canvas application. (see optional tracking tags)
  • For type 27 ads, you can use url_tags to specify a list of query parameters. These query parameters will be appended to the ad's URL, which points to an external website or a Facebook canvas application. (see optional tracking tags).

Obtaining Objects

Many ad creatives such as Sponsored stories and social ads for Facebook page require object_id of relevant Facebook object or URL of a page tab. You can obtain a list of object IDs using the Graph API with /{account-id}/connectionobjects, which returns the objects for the session user. Please see Obtaining ad account objects

Obtaining Page Posts

You can use Graph API for obtaining posts of a page (see Page):

To retrieve a list of page's own posts (for type 25 or type 27 ad) call

curl https://graph.facebook.com/<PAGE_ID>/posts?access_token=<ACCESS_TOKEN>

You can use any valid page access_token or user access_token.

Note: The ID of the post is returned in the following format: <Page_ID>_<Story_ID>. Only the "Story_ID" element should be supplied as the story_id.

Obtaining Page Events

You can use Graph API for obtaining events of a page (see Page):

To retrieve a list of page's own events (for type 25 ad) call

curl https://graph.facebook.com/<PAGE_ID>/events?access_token=<ACCESS_TOKEN>

You can use any valid access_token or user access_token.

Including social context in ads for external sites

Social Context may be applied to external URL (type 1) ads by means of the related_fan_page parameter. The social context can either be determined by Facebook or specified by you. Facebook can match the ad's external URL (specified in the link_url parameter) to a related Facebook page (or Facebook app) based on the validation criteria. Alternatively you can specify the Object Id of the Facebook Page (or Facebook App) you want the social context to be applied from. Facebook will still validate this Object ID matches the URL in the Ad.

Note: Pages must have a minimum of 10 fans before social context will be applied.

Value of related_fan_page Description
0 Indicates that no social context should be provided.
1 Indicates that Facebook should automatically match and include social context from a Facebook page (or Facebook app). Specifying 1 is equivalent to checking the "Show stories about people interacting with this page" checkbox in the Ad Manager web interface. This is the default value.
{object ID of a Facebook page or app} The object ID of a Facebook page (or Facebook app). If Facebook matches the object ID to the specified link_url value, Facebook will apply social context for the ad, based on the object ID.

Note: When a Facebook App Object ID is provided the Social Context of the app is used rather than the App Profile Page.

The following example creates an ad group with a related link_url value corresponding to the object ID. Facebook will provide social context for the ad, based on the object ID, if Facebook matches the object ID to the specified link_url value:

curl -F "campaign_id=6003417011234"
 -F "bid_type=1"
 -F "max_bid=30"
 -F "targeting={'countries':['US']}"
 -F  "creative=
   {'title':test_ad','body':'test_ad,
   'link_url':'http:\/\/www.cnn.com',
   'image_file':'test.jpg',
   'related_fan_page':5550296508}"
 -F "name=test" -F test.jpg=@.\test.jpg
 "https://graph.facebook.com
 /act_368811234/adgroups?access_token=_____"

The result will be of the standard ad creation response form.

{"id":"6003557111234"}

URL Validation

In order to receive the social context of a Page the url of the creative must be pointing to website listed in the related page's "about" section. Note, only one website may be listed in the "about" section for the validation to work. The urls are validated according to the following rules.

If the Page lists the URL "www.mybrand.com";

  • Ads pointing to www.mybrand.com will be eligible to receive social context.
  • Ads pointing to www.mybrand.com/?=query will also be eligible - our system will omit any query after a ? in the URL.
  • Ads pointing to www.mybrand.com/directory will not. Our system will not accept subdomains or directories within the domain listed.

If the Page lists the URL "www.mybrand.com/directory";

  • Ads pointing to www.mybrand.com/directory will be eligible to receive social context.
  • Ads pointing to www.mybrand.com/directory/?=query will also be eligible - our system will omit anything after a ? in the URL.
  • Ads pointing to www.mybrand.com/ will not be eligible.
