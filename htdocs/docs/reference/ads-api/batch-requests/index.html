Batch Requests on the Ads API.

The most efficient way to manage ads on the Ads API is via batched requests. This document outlines how the batched request functionality can be used to perform some of the more common requests in the Ads API.

Batched requests allow you to combine together a number of Graph API calls into the one HTTP request. This request is received by the Ads API and split up into its constituent requests. This makes batched requests the most efficient way of interacting with the Ads API.

For even greater efficiency batch requests may be made in parallel using separate processing threads.

Note: Each batched request can contain a maximum of 50 requests. For Ad group creation we advise creating 10 or fewer Ad groups per batch.

Ad Groups

Note: The batch requests for Ad groups and Ad campaigns are very similar so will not be dealt with separately here.

Creating many similar Ad Groups via a batched request

The following example shows how to create 3 Ad groups using 1 Ad Creative and 3 different targeting specs.

In this example the Ad Creative is defined first and then referenced in the creation of each Ad group 

curl -F 'access_token=______' 
  -F 'test1=@./test1.jpg'  
  -F 'batch=[
             {
              "method": "POST",
              "name": "create_creative",
              "relative_url": "act_187687683/adcreatives",
              "attached_files": "test1",
              "body": "title=Test title&body=Test body&link_url=http://www.test12345.com&image_file=test1.jpg"
             },
             {
              "method": "POST",
              "relative_url": "act_187687683/adgroups",
              "body": "campaign_id=6004163746239&redownload=1&bid_type=1&max_bid=30&creative={\"creative_id\":\"{result=create_creative:$.id}\"}&targeting={\"countries\":[\"US\"]}&name=test1"
             },
             {
              "method": "POST",
              "relative_url": "act_187687683/adgroups",
              "body": "campaign_id=6004163746239&redownload=1&bid_type=1&max_bid=30&creative={\"creative_id\":\"{result=create_creative:$.id}\"}&targeting={\"countries\":[\"GB\"]}&name=test2"
             },
             {
              "method": "POST",
              "relative_url": "act_187687683/adgroups",
              "body": "campaign_id=6004163746239&redownload=1&bid_type=1&max_bid=30&creative={\"creative_id\":\"{result=create_creative:$.id}\"}&targeting={\"countries\":[\"IE\"]}&name=test3"
             }
            ]' https://graph.facebook.com

Note: The batched request process uses the JSONPath expression format to reference the previous requests.

The response for this request includes individual response codes for each request as well as the standard Graph API response. Please see the batched request documentation for a detailed explanation of the batch response.

Creating different Ad Groups via a batched request

The following example shows how to create 3 Ad groups using 2 different images called test1.jpg and test2.jpg

curl -F 'access_token=____' 
  -F 'test1=@./test1.jpg' 
  -F 'test2=@./test2.jpg' 
  -F 'batch=[
             {
              "method": "POST",
              "relative_url": "act_187687683/adgroups",
              "attached_files": "test1",
              "body": "campaign_id=6004163746239&bid_type=1&max_bid=30&creative={\"title\":\"Test title 1\",\"body\":\"Test body 1\",\"link_url\":\"https://apps.facebook.com/testapp/\", \"image_file\":\"test1.jpg\"}&targeting={\"countries\":[\"US\"]}&name=test1"
             },
             {
              "method": "POST",
              "relative_url": "act_187687683/adgroups",
              "attached_files": "test2",
              "body": "campaign_id=6004163746239&bid_type=1&max_bid=30&creative={\"title\":\"Test title 2\",\"body\":\"Test body 2\",\"link_url\":\"https://apps.facebook.com/testapp/\", \"image_file\":\"test2.jpg\"}&targeting={\"countries\":[\"US\"]}&name=test2"
             },
             {
              "method": "POST",
              "relative_url": "act_187687683/adgroups",
              "attached_files": "test1",
              "body": "campaign_id=6004163746239&bid_type=1&max_bid=30&creative={\"title\":\"Test title 3\",\"body\":\"Test body 3\",\"link_url\":\"https://apps.facebook.com/testapp/\", \"image_file\":\"test1.jpg\"}&targeting={\"countries\":[\"US\"]}&name=test3"
             }
           ]' https://graph.facebook.com

The response for this request includes individual response codes for each request as well as the standard Graph API response. Please see the batched request documentation for a detailed explanation of the batch response.

Updating Ad Groups via a batch request.

Ad Groups can also be updated via a batch request. The following example updates the bids for 3 Ad groups in the one call.

  curl -F 'access_token=____' 
    -F 'batch=[
               {
                "method": "POST",
                "relative_url": "6004251715639",
                "body": "redownload=1&max_bid=35"
               },
               {
                "method": "POST",
                "relative_url": "6004251716039",
                "body": "redownload=1&max_bid=35"
               },
               {
                "method": "POST",
                "relative_url": "6004251715839",
                "body": "redownload=1&max_bid=35"
               }
              ]' https://graph.facebook.com

The response for this request includes individual response codes for each request as well as the standard Graph API response. Please see the batched request documentation for a detailed explanation of the batch response.

Note: By including redownload=1 in the relative URL you will receive back the full Ad group specification including the Ad group id to help identify which Ad groups were updated.

Note: To update the creative of an Ad group the entire creative must be specified or a new creative id must be supplied. This is because Ad Creatives cannot be edited after they have been created (except for name and run status).

Read Ad Groups

If you have a large number of Ad groups to read the best option is to split the request over multiple requests within a batched request. The following example shows a number of ways of requesting Ad groups in an efficient manner.

  curl -F 'access_token=____' 
    -F 'batch=[
               {
                "method": "GET",
                "relative_url": "?ids=6003356308839,6004164369439"
               },
               {
                "method": "GET",
                "relative_url": "6003356307839/adgroups"
               },
               {
                "method": "GET",
                "relative_url": "act_187687683/adgroups?campaign_ids=[6003356307839, 6004164259439]"
               }
              ]' https://graph.facebook.com

The response for this request includes individual response codes for each request as well as the standard Graph API response. Please see the batched request documentation for a detailed explanation of the batch response.

Note: In the above example 6003356308839 and 6004164369439 are Ad group ids and 6003356307839 and 6004164259439 are Ad campaign ids.

Ad Statistics

Note: The batch requests for stats for Ad groups and Ad campaigns are very similar so will not be dealt with separately here.

Reading Ad Group Statistics

If you have a large number of Ad statistics to read the best option is to split the request over multiple requests within a batched request. The following example shows a number of ways of requesting Ad statistics in an efficient manner.

  curl -F 'access_token=____' 
    -F 'batch=[
               {
                "method": "GET",
                "relative_url": "act_19643108/adgroupstats?adgroup_ids=[6003356308839,6004164369439]"
               },
               {
                "method": "GET",
                "relative_url": "6003356308839/stats"
               },
               {
                "method": "GET",
                "relative_url": "act_187687683/adgroupstats?campaign_ids=[6003356307839, 6004164259439]"
               }
              ]' https://graph.facebook.com

The response for this request includes individual response codes for each request as well as the standard Graph API response. Please see the batched request documentation for a detailed explanation of the batch response.

Note: In the above example 6003356308839 and 6004164369439 are Ad group ids and 6003356307839 and 6004164259439 are Ad campaign ids.

Note: For Ad accounts with a large number of Ad groups using act_<account_ID>/adgroupstats is not recommended as it may cause the request to timeout.

Estimates Reach

Querying reach estimates for an Ad Group

If you want to get the latest reach estimate for a large number of Ad groups the best approach is to request multiple reach estimates at once using a batched request. Up to 50 reach estimates may be requested in a single batched request.

The following example shows the reach estimate being requested for 3 different Ad groups at the one time:

  curl -F 'access_token=____' 
    -F 'batch=[
               {
                "method": "GET",
                "relative_url": "6003356308839/reachestimate"
               },
               {
                "method": "GET",
                "relative_url": "6004164369439/reachestimate"
               },
               {
                "method": "GET",
                "relative_url": "6004017499039/reachestimate"
               }
              ]' https://graph.facebook.com

The response for this request includes individual response codes for each request as well as the standard Graph API response. Please see the batched request documentation for a detailed explanation of the batch response.

Querying reach estimates for a targeting spec

If you want to get the latest reach estimate for a large number of targeting specs the best approach is to request multiple reach estimates at once using a batched requests. Up to 50 reach estimates may be requested in a single batched requests.

The following example shows the reach estimate being requested for 3 different targeting specs at the one time:

  curl -F 'access_token=____' 
    -F 'batch=[
               {
                "method": "GET",
                "relative_url": "act_600335/reachestimate?currency=USD&targeting_spec={'countries':['US']}"
               },
               {
                "method": "GET",
                "relative_url": "act_600335/reachestimate?currency=USD&targeting_spec={'countries':['FR']}"
               },
               {
                "method": "GET",
                "relative_url": "act_600335/reachestimate?currency=USD&targeting_spec={'countries':['US'],'zips':['94402']}"
               }
              ]' https://graph.facebook.com

The response for this request includes individual response codes for each request as well as the standard Graph API response. Please see the batched request documentation for a detailed explanation of the batch response.

ETags

ETags can be used with batch queries to check if anything has changed since the last time you made the batch request. For more on ETags please review the Using ETags documentation.
