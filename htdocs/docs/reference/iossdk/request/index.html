Request

SDK Reference › iOS SDK › Request

The API request methods used in the Facebook iOS SDK. These are defined in Facebook.h.

You implement these methods to do the following:

  • Make a server request using the Graph API.

  • Make a server request using REST API and FQL.

  • Make a server request using the Legacy REST API.

Instance Methods

requestWithParams:andDelegate:

Call this method to make a request to the REST API with specified parameters.

- (FBRequest*)requestWithParams:(NSMutableDictionary *)params
                   andDelegate:(id <FBRequestDelegate>)delegate;

Parameters

params - Key-value string pairs representing the API call parameters.

delegate - The receiver's delegate or nil if it doesn't have a delegate. See FBRequestDelegate for information on the delegate protocol.

Return Value

The FBRequest object corresponding to this API call.

Notes

One of the parameters to this call must be method and it's value must be a valid REST server API method. If you are using FQL your method parameter should be set to ''fql.query''. You should set up your delegate to handle the API results including error scenarios.

requestWithMethodName:andParams:andHttpMethod:andDelegate:

Call this method to make a request to the REST API with a given API method, specified parameters, and an HTTP method.

- (FBRequest*)requestWithMethodName:(NSString *)methodName
                   andParams:(NSMutableDictionary *)params
                   andHttpMethod:(NSString *)httpMethod
                   andDelegate:(id <FBRequestDelegate>)delegate;

Parameters

methodName - Valid REST API method.

params - Key-value string pairs representing the API call parameters.

httpMethod - HTTP method.

delegate - The receiver's delegate or nil if it doesn't have a delegate. See FBRequestDelegate for information on the delegate protocol.

Return Value

The FBRequest object corresponding to this API call.

Notes

You may use this method if you want control over the type of HTTP method used in the API call. You should set up your delegate to handle the API results including error scenarios.

requestWithGraphPath:andDelegate:

Call this method to make a request to the Graph API.

- (FBRequest*)requestWithGraphPath:(NSString *)graphPath
                   andDelegate:(id <FBRequestDelegate>)delegate;

Parameters

graphPath - The path to the Graph API endpoint. For example, to fetch data about the currently logged in user this parameter should be ''me'', representing a call to the https://graph.facebook.com/me endpoint.

delegate - The receiver's delegate or nil if it doesn't have a delegate. See FBRequestDelegate for information on the delegate protocol.

Return Value

The FBRequest object corresponding to this API call.

Notes

You should set up your delegate to handle the API results including error scenarios.
