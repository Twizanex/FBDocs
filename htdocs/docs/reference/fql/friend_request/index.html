friend_request

Advanced Topics › FQL › friend_request

An FQL table which shows the pending friend requests sent by or to the current session user.

To read the friend_request table you need:

  • read_requests permissions for any friend requests that are sent by or to the current session user.

Note: You can only query this table when:

  • uid_to is specified as the current session user or
  • uid_from is specified as the current session user and uid_to is a particular user (if no request exists, this will return a blank array)

If the current session user has hidden a friend request it will not be returned in this table.

Examples

All pending friend requests sent to the current session user (Try this query)

SELECT uid_from, time, message FROM friend_request WHERE uid_to = me()

A pending friend request sent by the current session user to another user

SELECT time, message FROM friend_request WHERE uid_from = me() AND uid_to = "204686"

コラム

インデックス可能 名前 タイプ 説明
* uid_to string

The ID of the user receiving the friend request.
uid_from string

The ID of the user making the friend request.
time time

A unix timestamp indicating when the friend request was sent.
message string

An optional message sent by the user with the friend request.
unread bool

Returns true if the user has not yet seen the friend request. Only available if the uid_to parameter is set to the logged-in user
