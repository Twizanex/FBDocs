unified_thread_action

Advanced Topics › FQL › unified_thread_action
Please note: We are in the process of making the new messages system available to all users. We are providing early access to this API for registered developer accounts only until the new messaging system is broadly available. You should use the message and thread tables for production applications at the current time.

This table should be used to access information about Add People and Leave Conversation actions performed on a thread in the new Facebook messaging system and is only applicable to group threads with three or more participants.

Access to this table requires the read_mailbox extended permission.

Example

Select all actions for a thread:

SELECT ... FROM unified_thread_action WHERE thread_id=...

action_id string

A unique version identifier that is increasing in the same order as the actions that have been performed.
actor object

The user who performed the action. This field is an object with properties name (string), email (string) and user_id (string).
* thread_id string

Unique identifier of the thread.
timestamp string

Time at which the action was performed (Unix-type timestamp with millisecond resolution).
type int

Flag to indicate whether the action is a subscribe (1) or unsubscribe (2).
users array

List of users on whom the action was performed. Each element of the list is an object with properties name (string), email (string) and user_id (string).
