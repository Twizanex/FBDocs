family

Advanced Topics › FQL › family

Detailed information about a user's family as represented in FQL.

In the Graph API the family information of a user can be found using the family connection of the user object.

To read the family table you need

  • an access token with basic permissions
  • the extended permission 'user_relationships'

Examples

Use this query to retrieve a list of all family members listed for the user. (Try this query)

SELECT profile_id, relationship FROM family WHERE profile_id = <uid>

Use this query to retrieve a list of all family members that are not linked to actual profiles. (Try this query)

SELECT uid, name, birthday FROM family WHERE profile_id = <uid>

コラム

インデックス可能 名前 タイプ 説明
* profile_id string

The ID of the user being queried.
uid string

The user ID of the relative, which is returned if the relationship is confirmed and links to another profile.
name string

The name of the relative, which is returned if the relationship is not linked to another profile.
birthday string

The birthday of the relative, which is returned if the relationship is not linked to another profile.
relationship string

A string describing how the relative is related to the user being queried.
