permissions

Advanced Topics › FQL › permissions

A representation of the permissions granted to your app by a User or Page represented in FQL.

The User object in the Graph API has an equivalent permissions connection which is an associative array of the granted permissions.

The permissions granted to your app by a User can also be subscribed to via real time updates in the Graph API.

To read the permissions table you need one of

  • a user access_token for a user of your app.
  • an app access_token for your app (described under App Login on the Authentication page.)

Usage Note

The 'Columns' list below is abbreviated. In addition to the uid column, there is one virtual column in this table for each permission which is available to be requested by applications. A common use of this table is checking which of a set of desired permissions have been granted to your application.

Your queries should be of the form

select <permissions> from permissions where uid = me()

where <permissions> is a comma separated list of the permissions you wish to check.

A full list of permissions can be found in the permissions documentation.

Example

To see if the manage_pages and / or publish_stream permissions were granted to your app by the current user: (Try this query)

select manage_pages, publish_stream from permissions where uid = me()
