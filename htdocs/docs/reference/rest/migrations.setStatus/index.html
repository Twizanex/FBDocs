migrations.setStatus

Advanced Topics › Legacy REST API › migrations.setStatus
Please note: We are in the process of deprecating the REST API, and have added equivalent support to the Graph API for this method. Please use the migrations field on Application object.

Configures the migration setting for a Platform migration. To get the value for migration, call migrations.getStatuses. You must sign your call with your application secret. Applications that should not include their application secret in their code (such as desktop applications), should not use this call.

パラメータ

名前 タイプ 説明
api_key string

The application key associated with the calling application.
call_id float

The request's sequence number. Each successive call for any session must use a sequence number greater than the last. We suggest using the current time in milliseconds, such as PHP's microtime(true) function. If you specify the call ID in your client, you don't need to pass it with every call.
sig string

An MD5 hash of the current request and your secret key, as described in the authentication guide.
v string

This must be set to 1.0 to use this version of the API.
format string

The desired response format, which can be either XML or JSON. (Default value is XML.)
callback string
