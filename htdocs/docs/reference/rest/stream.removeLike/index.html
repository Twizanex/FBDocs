stream.removeLike

Advanced Topics › Legacy REST API › stream.removeLike
Please note: We are in the process of deprecating the REST API, so if you are building a new application you shouldn't use this function. Instead use the Graph API and issue a DELETE to the Likes connection of the Post object for the given User object

This method removes a like a user added to a post or comment.

Desktop applications must pass a valid session key, and can remove likes made by the user associated with that session key only. Other applications can remove any likes, provided you have a valid post_id or comment_id.

In order to remove a Like from a post or comment, the user must grant your application the publish_stream extended permission.

パラメータ

名前 タイプ 説明
uid string

The user ID of the user who liked the post or comment. If this parameter is not specified, then it defaults to the session user.

Note: This parameter applies only to Web applications and is required by them only if the session_key is not specified. Facebook ignores this parameter if it is passed by a desktop application.
post_id string

The ID of the post.
comment_id string

The ID of the comment.

レスポンス

This method returns true on success, or false and an error code if it fails.
