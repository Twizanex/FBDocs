Games Monetization

Getting Started › Apps on Facebook › Games Monetization

Most games on Facebook Platform monetize through a freemium or Free to Play model and the use of micro-transactions, powered by Facebook Credits.

Games can either implement their own virtual currency which can be purchased via Facebook Credits or use Facebook Credits as their in-game virtual currency directly. This virtual currency is then used to purchase various types of virtual goods. Either way developers can use Facebook Credits to monetize their games internationally via 80+ payment methods supported in 50+ countries around the world.

Pro-Tip : Most developers are familiar with the default "Credits" implementation, but fewer developers are familiar with the "Credits as a Payment" flow, which might work better if you have already invested in your own in-game currency, or have other reasons for maintaining your own in-game currency (such as having a cross-platform app). You can learn more about payment APIs in our Facebook credits docs and Facebook credits best practices.

Next, we list a few common examples of virtual goods usage in the Facebook platform.

Vanity Items

Vanity Items are in game virtual goods that have no direct effect on game play. Generally these are cosmetic items that can be used to enhance a players visible avatar, house, base, town, etc.

An important motivator for purchasing vanity items is scarcity or uniqueness. This can be enhanced by implementing limited time sales or limiting the quantity of an item sold. Should the item prove to be popular it can be re-introduced into the ecosystem at later points in the games lifecycle as a slight variant on the original item, so as to preserve the uniqueness of the original purchasers item.

Functional Items and Buffs

Functional Items are the opposite of Vanity Items. These are items that can be purchased in-game that have a direct effect on gameplay - a faster car or more powerful weapon.

Buffs are consumable items (limited number of uses) that can be applied to a player’s game state increasing their ability to perform certain tasks. For example a Match Three game can sell an item that can be used once and increases a combo multiplier for the current session allowing the player to achieve a greater score.

It is important to think carefully about how to use Buffs in your game if you are building a skill-based game. It is important that buffs don't become the way to cheat but are a part of game play.

Session Extensions (Speed Ups and Energy Packs)

These are items commonly sold in Social Games that increase session length or decrease the amount of down time between sessions.

A common mechanic is to limit session length by introducing a limited use but recharging resource. All actions performed in the game require some amount of this resource and once the player runs out they will need to wait for this to recharge, which happens slowly over time. Items can be purchases to give a quick boost of energy allowing the player to keep player during the same session.

Another common mechanic is to introduce the concept of real-time effecting the completion of activities. This is usually found in city management or empire building games where players need to train troops or construct buildings. The activity requires X amount of time to complete. If players want to expedite the construction they can speed up the process by spending a certain amount of virtual currency.

Setting a fixed number of moves or plays per session is another method to throttle player consumption of content and allow for monetization opportunities. This is a similar to tokens in a real world arcade. Each day the player receives X tokens. Once these tokens have been spent the player must return the following day, or purchase additional tokens to keep playing.

Next steps

To get started, please familiarize yourself with the following technologies as, in most cases developers will need to interact with them:

  • JavaScript SDK - a thin wrapper on the Dialog API and Graph API, used to display UI elements for Facebook apps.
  • Graph API - a RESTful API for interacting with the Social Graph.
  • Credits API - an API to implement payments with the Facebook Platform. This is required for Canvas games.
