與用戶通訊

串流頻道並非是讓應用程式用來向用戶發佈每日更新等客戶通訊目的而設，而是讓用戶與朋友分享有趣的相關內容。我們建立了特定的產品，提供給應用程式作為適合應用程式直接與用戶群通訊的渠道。以下所列為「適用於應用程式向用戶發送」的通訊工具範例。

應用程式產生的要求

應用程式產生的要求與用戶向用戶發送的要求有所不同，應用程式能透過可更新書籤上的通知計數的 Graph API 產生發送要求。這是帶動不斷的交流互動的好方法，並能有效提醒用戶作出反應行動，如移動棋子、管理農場，或檢視體育賽事得分、星座及新聞文章等每日更新。

注意：此功能僅供畫布應用程式使用。

應用程式專頁

使用應用程式專頁與用戶進行通訊是用來帶動用戶與應用程式重新互動的好方法，同時可確保用戶收到應用程式的更新通知，如新關卡等級或新品上線等訊息。與 Facebook 專頁類似，管理員可在用戶留言上作出回應、上傳相片並直接與用戶進行互動。

Open Graph 頁面

在您的網頁上包括 Open Graph 標籤，讓您的網站頁面具有等同於 Facebook 專頁的效能。這表示，當用戶在您的網站頁面上點擊了「讚」按鈕，您的頁面與用戶之間便會產生聯繫。您的頁面將回出現在用戶個人檔案的「愛好及興趣」部分，而您可向用戶發佈更新，更新也會顯示在用戶的動態消息中。

電郵

電郵是應用程式向用戶發送通訊的最佳溝通方法。您可透過電郵進階權限要求取得用戶電郵，或者讓用戶直接在您提供的欄位中向您提供他們的電郵地址。
