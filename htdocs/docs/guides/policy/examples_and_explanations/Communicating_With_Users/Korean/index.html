사용자들과 소통하기

스트림 채널은 앱이 사용자들에 대한 일일 업데이트 제공하는 등 사용자와 소통하기 위한 것이 아니라 사용자들이 친구들과 흥미있고 유의미한 콘텐츠를 공유하기 위한 것입니다. 저희는 앱이 사용자들과 적절한 방법을 통해 직접 소통할 수 있도록 해주는 별도의 서비스를 구축했습니다. 아래 예들이 '앱이 사용자들과' 소통하는 툴입니다.

앱이 생성한 요청

사용자 대 사용자 요청과는 달리 앱이 생성한 요청은 북마크 수를 업데이트하는 그래프 API를 통해 앱이 처음 시작할 수 있습니다. 이는 사용자들의 앱과의 반복 소통을 독려하고 사용자들이 체스 말 옮기기나 농장 관리, 또는 운동 경기 결과, 별자리, 새 기사와 같은 일일 업데이트 보기와 같은 행동을 취하도록 유도하는 좋은 방법입니다.

참고: 이 기능은 캔버스 앱에서만 사용가능합니다.

앱 페이지

사용자들과의 소통을 위해 앱 페이지를 이용하는 것은 앱과의 반복 소통을 유도하고 사용자들이 새로운 단계나 구입 아이템과 같은 앱의 업데이트를 계속 받도록 해주는 좋은 방법입니다. Facebook 페이지와 마찬가지로 관리자는 사용자들의 게시물에 댓글 달기, 사진 업로드, 사용자와 직접 대화를 할 수 있습니다.

오픈 그래프 페이지

회원님의 웹 페이지에 오픈 그래프 태그를 추가하면 회원님의 페이지가 Facebook 페이지와 비슷해집니다. 즉, 사용자가 회원님의 페이지 상에서 좋아요 버튼을 클릭하면 회원님의 페이지와 사용자 사이에 연결 관계가 형성됩니다. 회원님의 페이지는 사용자의 프로필에서 '좋아하는 것 및 관심사' 섹션에 표시되며, 회원님은 사용자에게 업데이트를 제공할 수 있고 이 업데이트는 사용자의 뉴스피드에 표시됩니다.

이메일

이메일은 앱과 사용자 사이의 소통에 있어 가장 최선의 방법입니다. 회원님은 이메일 정보 사용 허가(email extended permission)를 통해서 또는 사용자에게 이메일 주소를 입력할 수 있는 공간을 직접 제공함으로써 사용자의 이메일 주소를 요청할 수 있습니다.
