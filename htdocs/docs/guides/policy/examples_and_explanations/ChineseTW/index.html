範例及解釋

我們的原則及政策內提供了所需項目的說明，而這些範例及解釋將會定期更新，助您了解我們的政策是如何反應在實際應用程式的功能。以下僅供說明用途，目的不是要限制 Facebook 開放平台政策的普用性。

與用戶通訊

我們建立了特定的產品，提供給應用程式作為適合應用程式直接與用戶群通訊的渠道。以下所列為「適用於應用程式向用戶發送」的通訊工具範例。
進階權限

使用進階權限為用戶打造豐富的社交體驗時要考慮的最佳實作方法。

相片

相片產品的使用注意事項助您避免遇到違反政策和收到用戶負面意見的問題。
禁用功能

Facebook 開放平台是開放給所有的開發人員，不過我們限制了某些功能以便保障用戶體驗。

獎賞用戶

雖然我們的政策禁止直接使用社交渠道提供獎勵，例如以獎賞為誘因要求用戶因此發佈串流動態，不過我們會允許特定類型的獎勵。

社交渠道

使用 Facebook 社交渠道，尤其是動態流方面要考慮的最佳實作方法。

用戶意見

這些訊號能助您更清楚了解用戶與其朋友對應用程式及其內容的評價。這些意見亦有助我們在開放平台上自動化及手動執行批准應用程式上線的決定，因為監控用戶體驗有助我們了解哪些應用程式向用戶提供了正面或負面的氣氛，影響平台氣氛。

品牌

您在設計圖標及選擇應用程式名稱時，請遵循有關指南，避免混淆用戶或損害 Facebook 品牌資產。
