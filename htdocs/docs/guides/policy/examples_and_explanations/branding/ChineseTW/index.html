非 Facebook 平台的 Facebook 應用程式指南

下列指南可以幫助您明確表明您的應用程序與 Facebook 相容，同時能幫助您建立自己的品牌識別，以及避免混淆用戶或損害 Facebook 品牌資產。如需要有關 Facebook 品牌指南的完整資料，請瀏覽我們的品牌授權中心。

應用程式名稱

在 Facebook 以外的地方，您可能想要明確指出您的應用程式與 Facebook 相容。但您不能以任何手法暗示這是 Facebook 官方或背書支持的應用程式。

只要您的應用程式名稱符合以下條件， 您可以使用「（您的應用程式名稱）for Facebook」的方法作命名：(1) 是獨特及原創的，以及 (2) 可直接清楚地知道該應用程式是由您作為開發人員所提供，而非 Facebook 所提供、贊助或支持的應用程式。

範例：

可接受的：

  • 可以接受 「Zwuitlite for Facebook」 ，因為 「Zwuitlite"」是獨特及原創的名稱。
  • 如果您叫 Bob，也可以使用 「Bob's AwesomeApp for Facebook」，因為您的名字讓買家知道應用程式由您所提供，而非來自 Facebook。

不能接受的:

  • 不可以接受「"Facebook 桌面」，因為「桌面」一詞太普通，而且名稱開頭不得使用 「Facebook」。
  • 不可以接受 「Pages for Facebook,」、「Video+ for Facebook」 或 「Photos for Facebook」，因為 「Pages」、「Video+」 及 「Photos」 均屬於描述應用程式及 Facebook 功能的詞彙。

名稱加上「加值」、「+」、「輕型版」、「推送」 或類似字眼，並不會使應用程式名稱顯得獨特。

  • 不可以接受 「FaceMessages for Facebook」 及 「VideoBook for Facebook」，因為 「Face」 和 「Book」 屬於 Facebook 商標的獨特元素。
  • 不可以接受 「FB 影片」 和 「F影片」，因為在這個情況中「F」 及 「FB」 很明顯是代表 Facebook。

圖標

您應該為您的應用程式設計可以代表自己品牌的圖標。應用程式圖標中，不可使用或合併 Facebook 的 F 標誌，或任何由 F 標誌所更改或衍生出來的元素。

您不能使用 FACEBOOK 標誌。您也不能使用 Facebook 其他圖形或圖標，例如我們著名的剪影標誌。

Open Graph「讚」行動

您可透過「讚」一詞建立自訂的開放社交關係圖（Open Graph） 行動，但該行動不可模仿或抄襲 Facebook 的「讚」按鈕設計。該行動不可混淆用戶，讓他們以為您的開放社交關係圖行動最後會與 Facebook 有所連結。

在您的開放社交關係圖行動中，切勿使用「豎起大拇指」圖像和「讚」一詞。
