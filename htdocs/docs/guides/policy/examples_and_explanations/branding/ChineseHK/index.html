非 Facebook 平台的 Facebook 應用程式指引

下列指引有助您準確表明您的應用程序與 Facebook 相容，同時能幫助您建立自己的品牌識別，以及避免混淆用戶或損害 Facebook 品牌資產。如需要有關 Facebook 品牌指引的完整資訊，請瀏覽我們的品牌授權中心。

應用程式名稱

您可能希望在 Facebook 以外的地方準確表明您的應用程式與 Facebook 相容。但您不能以任何手法暗示這是一個正式或由 Facebook 支持的應用程式。

只要您的應用程式名稱 (1) 是獨特及原創的，以及 (2) 可直接清楚地知道應該程式由您作為開發人員所提供，而非 Facebook 所提供、贊助或支持的應用程式，您可以使用「（您的應用程式名稱）for Facebook」的方法作命名。

範例：

可接受的：

  • 可以接受 "Zwuitlite for Facebook" ，因為 "Zwuitlite" 是獨特及原創的名稱。
  • 如果您叫 Bob，便可以使用 "Bob's AwesomeApp for Facebook"，因為您的名字讓買家知道應用程式由您所提供，而非 Facebook。

不能接受的:

  • 不可以接受 "Facebook Desktop"，因為「桌面」一詞太普通，而且不可以 "Facebook" 作名稱開首。
  • 不可以接受 "Pages for Facebook,"、"Video+ for Facebook," 或 "Photos for Facebook"，因為"Pages"、"Video+" 及 "Photos" 均屬於描述應用程式及 Facebook 功能的詞彙。

加上"plus,"、"+,"、"lite,"、"with push" 或類似字眼不會使應用程式名稱變為獨特。

  • 不可以接受 "FaceMessages for Facebook" 及 "VideoBook for Facebook"，因為 "Face" 和 "Book" 屬於 Facebook 商標的獨特元素。
  • 不可以接受 "FB Video" 和 "FVideo"，因為在這個情況中 "F" 及 "FB" 很明顯是代表 Facebook。

圖標

您應該為應用程式設計圖示以代表品牌。應用程式圖示中，不可使用或合併 Facebook 的 F 標誌，或任何改動或由 F 標誌所衍生的元素。

您不能使用 FACEBOOK 標誌。您也不能使用 Facebook 其他圖形或圖示，例如我們著名的剪影標誌。

Open Graph「讚好」行動

您可透過「讚好」一詞建立自訂的 Open Graph 行動，但該行動不可模仿或抄襲 Facebook 的讚好按鈕設計。該行動不可混淆用戶，使他們以為您的 Open Graph 行動最後會與 Facebook 有所連結。

在您的 Open Graph 行動中，切勿使用「豎起大拇指」圖像和「讚好」一詞。
