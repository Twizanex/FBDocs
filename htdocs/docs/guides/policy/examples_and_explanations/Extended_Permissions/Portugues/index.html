Exemplos e explicações: Permissões estendidas

Número de permissões

Você deve solicitar somente as permissões de que precisa para operar o seu aplicativo. Na verdade, nossas análises mostram uma queda na taxa de cliques se o aplicativo apresenta mais de três permissões ao mesmo tempo.

Contexto

Como uma prática recomendada, você só deve solicitar permissões estendidas quando razoável, quando o usuário se envolver com recursos que exijam seu uso. Por exemplo, se você está pedindo ao usuário uma permissão publish_stream para poder criar uma interface de usuário de compartilhamento personalizada, o usuário irá entender melhor o contexto por trás da solicitação, se essa for apresentada com a permissão enquanto interage com a funcionalidade de compartilhamento do aplicativo.
