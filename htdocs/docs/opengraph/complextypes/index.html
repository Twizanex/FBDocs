Complex Types

Core Concepts › Open Graph › Complex Types

Complex Types are a set of predefined groups of meta tags that can be used as a property of any Open Graph object. Apps should use these types for properties where possible to help the display and aggregation of actions and objects. The following Complex Types are available:

  • Audio
  • ContactInfo
  • GeoPoint
  • Image
  • Video

As these are not object types, there are no instances of a complex type. Rather, the values for a complex type are contained in the object that uses them.

The samples below assume that you've created a custom Open Graph object-type Restaurant named your-og-app:restaurant. In each sample, the Restaurant object-type is extended with a custom property of type complex. The application's developer has to assign the new property a name, which will then be used as the property's meta tag (example: your-og-app:podcast of type Audio).

Audio

Use the Audio complex type when defining objects that contain audio information (e.g. podcast, music, etc.).

Property Name Description Type
URL URL of the audio resource URL
Secure URL Secure URL of the audio resource URL
Type MIME type of the audio resource String

Example

<html>
  <head prefix="og: http://ogp.me/ns# fb: http://ogp.me/ns/fb# 
                your-og-app: http://ogp.me/ns/apps/your-og-app#">
  <meta property="fb:app_id"                      content="YOUR_APP_ID"> 
  <meta property="og:type"                        content="your-og-app:restaurant"> 
  <meta property="og:title"                       content="Sample Restaurant"> 
  <meta property="og:description"                 content="A great sample restaurant"> 
  <meta property="og:image"                       content="https://your-great-image">
  <meta property="your-og-app:podcast:url"        content="http://my-audio.mp3">
  <meta property="your-og-app:podcast:secure_url" content="http://my-audio.mp3">
  <meta property="your-og-app:podcast:type"       content="audio/mp3"> 
</head>
<body>
    <!--a wonderful web page of Sample Restaurant -->
</body>
</html>

ContactInfo

Use the ContactInfo complex type when defining objects that specify address, phone number or web site information.

Property Name Description Type
Street Address Physical address String
Locality City or municipality name String
Region State or province String
Postal Code Zip or other postal identifier String
Country Name Valid country or nation String
Email Address Valid email address String
Phone Number Valid phone number with area code String
Fax Number Valid phone number with area code String
Website Valid URL to website URL

Example

<html>
  <head prefix="og: http://ogp.me/ns# fb: http://ogp.me/ns/fb# 
       your-og-app: http://ogp.me/ns/apps/your-og-app#">
  <meta property="fb:app_id"                    content="YOUR_APP_ID"> 
  <meta property="og:type"                      content="your-og-app:restaurant"> 
  <meta property="og:title"                     content="Sample Restaurant"> 
  <meta property="og:description"               content="A great sample restaurant"> 
  <meta property="og:image"                     content="https://your-great-image"> 
  <meta property="your-og-app:owner:street_address" 
        content="1601 S. California St."> 
  <meta property="your-og-app:owner:locality"       content="Palo Alto"> 
  <meta property="your-og-app:owner:region"         content="California"> 
  <meta property="your-og-app:owner:postal_code"    content="94304"> 
  <meta property="your-og-app:owner:country_name"   content="United States"> 
  <meta property="your-og-app:owner:email"          content="nospam@fb.com"> 
  <meta property="your-og-app:owner:phone_number"   content="111-111-1111"> 
  <meta property="your-og-app:owner:fax_number"     content="111-111-1111"> 
  <meta property="your-og-app:owner:website"    
        content="http://www.facebook.com">
</head>
<body>
    <!--a wonderful web page of Sample Restaurant -->
</body>
</html>

GeoPoint

Use the GeoPoint complex type when defining objects that specify spatial information (e.g. geographic location).

The Facebook platform currently uses the GeoPoint complex type to render maps. For example, Newsfeed and Ticker stories are rendered using a map view instead of an image if the object has defined GeoPoint custom properties. Also, you can build Map Layout Style Timeline aggregations that are more relevant to your application by leveraging this type.

Property Name Description Type
Latitude Latitude of location Float
Longitude Longitude of location Float
Altitude Altitude of location Float

Example

<html>
  <head prefix="og: http://ogp.me/ns# fb: http://ogp.me/ns/fb# 
               your-og-app: http://ogp.me/ns/apps/your-og-app#">
  <meta property="fb:app_id"                      content="YOUR_APP_ID"> 
  <meta property="og:type"                        content="your-og-app:restaurant"> 
  <meta property="og:title"                       content="Sample Restaurant"> 
  <meta property="og:description"                 content="A great sample restaurant"> 
  <meta property="og:image"                       content="https://your-great-image"> 
  <meta property="your-og-app:location:latitude"  content="37.416382"> 
  <meta property="your-og-app:location:longitude" content="-122.152659"> 
  <meta property="your-og-app:location:altitude"  content="42">
</head>
<body>
    <!--a wonderful web page of Sample Restaurant -->
</body>
</html>

Image

Use the Image complex type when defining objects which contain more than one image.

Property Name Description Type
URL URL for the image URL
Secure URL Secure URL for the image URL
Type MIME type of the image String
Width Width of image in pixels Integer
Height Height of image in pixels Integer

Example

<html>
  <head prefix="og: http://ogp.me/ns# fb: http://ogp.me/ns/fb# 
                your-og-app: http://ogp.me/ns/apps/your-og-app#">
  <meta property="fb:app_id"                   content="YOUR_APP_ID"> 
  <meta property="og:type"                     content="your-og-app:restaurant"> 
  <meta property="og:title"                    content="Sample Restaurant"> 
  <meta property="og:description"              content="A great sample restaurant"> 
  <meta property="og:image"                    content="https://your-great-image"> 
  <meta property="your-og-app:menu:url"        content="http://menu.png"> 
  <meta property="your-og-app:menu:secure_url" content="https://menu.png"> 
  <meta property="your-og-app:menu:type"       content="image/png"> 
  <meta property="your-og-app:menu:width"      content="200"> 
  <meta property="your-og-app:menu:height"     content="500">
</head>
<body>
    <!--a wonderful web page of Sample Restaurant -->
</body>
</html>

Video

Use the Video complex type when defining objects which contain videos.

Property Name Description Type
URL URL for the video URL
Secure URL Secure URL for the video URL
Type MIME type of the video String
Width Width of video in pixels Integer
Height Height for video in pixels Integer

Example

<html>
  <head prefix="og: http://ogp.me/ns# fb: http://ogp.me/ns/fb# 
                your-og-app: http://ogp.me/ns/apps/your-og-app#">
  <meta property="fb:app_id"                         content="YOUR_APP_ID"> 
  <meta property="og:type"                           content="your-og-app:restaurant"> 
  <meta property="og:title"                          content="Sample Restaurant"> 
  <meta property="og:description"                    content="A great sample restaurant"> 
  <meta property="og:image"                          content="https://your-great-image"> 
  <meta property="your-og-app:commerical:url"        content="http://welcome.here.mp4"> 
  <meta property="your-og-app:commerical:secure_url" content="https://welcome.here.mp4"> 
  <meta property="your-og-app:commerical:type"       content="video/mp4"> 
  <meta property="your-og-app:commerical:width"      content="100"> 
  <meta property="your-og-app:commerical:height"     content="200">
</head>
<body>
    <!--a wonderful web page of Sample Restaurant -->
</body>
</html>
