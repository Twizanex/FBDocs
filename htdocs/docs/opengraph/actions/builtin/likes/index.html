Built-in Like

Core Concepts › Open Graph › Built-in Like

Like, one of the most ubiquitous actions by Facebook users on the web today, is now available as a built-in Open Graph action. Developers can now design how Like, as an Open Graph action, fits into the core user experience of any app, through building a custom in-app “like button” that works across any mobile and web platform, helping to improve in-app content relevance while enjoying additional distribution on Facebook.

Publishing Built-in Likes

An app can publish a built-in Like action, on behalf of the user, as long as the following conditions are true:

  1. The viewer of the in-app content is a Facebook user who has Facebook-authed and granted the app publish_actions permission
  2. The in-app content has an Open Graph object page that is properly marked-up using Open Graph metatags
  3. The viewer has intentionally clicked on an in-app “like button” associated with the in-app content

Here, developers should design their apps to provide an in-app “like button” that could be clicked on. Apps with existing in-app “like buttons” can now benefit from additional distribution on Facebook when these buttons publish built-in Like action stories to Open Graph.

A successful publish of a built-in Like action will appear as an Open Graph story in all places that other Open Graph action stories appear (Ticker, News Feed, Timeline, Activity Log).

To start configuring an app to have the ability to publish a built-in Like action on an Open Graph object:

  1. Within the Open Graph section of the Developer App configuration, add the built-in Like action, which should appear in the drop-down when adding a new Open Graph action.

  2. Create a custom in-app “like button” associated with each Open Graph object that users can “like”.
  3. When a user clicks on this button and has both FB-authed and provided the `publish_actions` permission to an app in the past, the app should publish a built-in Like action story on the Open Graph object.

To publish a built-in Like action on an Open Graph object, invoke the following HTTP POST request with a user’s access token and the url of the Open Graph object. This Open Graph object can be of any type.

curl -X POST
     -F 'access_token=USER_ACCESS_TOKEN' \
     -F 'object=OG_OBJECT_URL' \
     https://graph.facebook.com/[User FB ID]/og.likes

which in turn will return the ID of the built-in Like action instance if the call is successful

{ “id”:”396528780387583” }

Note that trying to re-publish a built-in Like action on an Open Graph object that has already been “liked” by the user in the past will result in the following error

{
  "error": {
    "message": "(#3501) User is already associated to the object
object on a unique action type Like. Original Action ID:
396497300390731",
    "type": "OAuthException",
    "code": 3501
  }
}

A re-publish will however succeed if the last known successful built-in Like action instance by the user on the object was removed. To undo a published “Like” action, essentially emulating an “Unlike”, invoke the following HTTP DELETE call on the ID of a Like action instance that was returned from a successful publish

curl -X DELETE
     -F 'access_token=USER_ACCESS_TOKEN' \
     https://graph.facebook.com/[Like Action Instance ID]

Notifications for Built-in Likes

The built-in Like action is particularly beneficial for apps where users can publish back content they have created via the Open Graph. Here, we call such an Open Graph action a “creation action”. For example, imagine a social cookbook app, in which a user can “write” (action) a “recipe” (object), and this activity is also published to Open Graph and shared with a user’s friends. Here, “write” is a “creation action” since each time this action is undertaken by the user, a new user-generated content, a “recipe” object, is created.

When a friend views the user’s “recipe” within the context of the social recipe app, and clicks on an associated “like button” that also publishes a built-in Like action to Open Graph, a Facebook notification would be generated. The user who created this “recipe” object through the “write” action will receive a Facebook notification that his/her friend “liked” the user’s “recipe”. This Facebook notification will only appear to a user when the person who “liked” the object is the user’s Facebook friend, this friend’s app activity is visible to the user, and the Open Graph object was “created” by the user through a “creation action”.

To enable Facebook Notifications when publishing built-in Like actions, configure a "creation action" for the object representing the user-generated content:

  1. Within the Open Graph section of the Developer App configuration, click to edit the Open Graph object that will represent the user-generated content. Then, click on the “Advanced” link to expand it.
  2. Fill in the “Creation Action” field with the Open Graph action that “creates” this object.

  3. Click on the “Save Changes” button.
  4. Whenever a Facebook user of an app “creates” user-generated content, the app should publish to Open Graph as usual, with the “creation action” as the Open Graph action and the user-generated content as the Open Graph object.

As outlined in the previous section, developers should design an in-app “like button”. When someone views user-generated content through an app and clicks on the associated in-app “like button”, publish a built-in Like action story to Open Graph, using the REST calls and conditions outlined earlier.

If the Open Graph object does not have an associated “creation action”, then no Facebook notifications will be generated when a built-in Like is published on it.
