Define Objects

Core Concepts › Open Graph › Define Objects

Objects should represent "entities" that users can act on in your application. Before an app can publish objects to Open Graph, object types, which are categories of "entities", must first be defined and registered by the app. Facebook also provides many object types built-in to Open Graph that are ready to use. Where applicable, you should use Built-in Object Types as it allows Facebook to help aggregate and boost relevant stories in the News Feed and Timeline.

  • Define Object Types for your app
  • Configure Custom Object Types with default properties
  • Define Custom Properties for Custom Object Types
  • GeoPoint Custom Properties for Custom Object Types
  • Video Custom Properties for Custom Object Types

Define Object Types

To get started, you can define and register object types in the Open Graph section of the App Dashboard. To define an object type, click on the "Create New Object Type" button in the Dashboard view and enter a "noun" which describes a high-level object users can act on within your app.

  • Use Built-in Object Types where appropriate. There are more opportunities for generating interesting aggregations by Facebook in the News Feed and Timeline.
  • Create objects that refer to other objects instead of a flat object structure. To see the analytics for a specific property, make that property refer to an object instead of being a simple type. Analytics are exposed at the object-level only. For example, when creating the custom object type "Restaurant" with a custom property “Cuisine”, to understand how users are sharing certain cuisine types, you should create “Cuisine” as a custom object type and make it a custom property of the "Restaurant" object.

Configure Custom Object Types

Once you have defined the name of a custom object type, you are ready to configure its properties.

  1. Basic Information:
    1. Name field is pre-filled with the object type you defined in the initial step.
  2. Object Properties:
    1. Title field can filled to specify the title for your sample object. You can preview how this will be displayed using the In News Feed box.
    2. Description field can filled to specify the description of your sample object. You can preview how this will be displayed using the In News Feed box.
    3. Image field can be one or more images. You can preview how this will be displayed using the In News Feed box. To create an array of images, construct multiple <meta og:image> tags.
    4. URL field points to the web page representing your sample object instance. It has all the required Open Graph tags.
    5. Add Another Property allows you to add additional custom properties to define your custom object. For example, for a "Dish" custom object, you may add reference to another custom object "Ingredient".
    6. In News Field box displays the News Feed story and helps you visualize your sample object.
  3. Singular and Plural Nouns:
    1. Singular Noun field is pre-filled with the singular form of the noun corresponding to your initial object type. This will be used when displaying an aggregate story in a user Timeline. Note that the article, or "determiner", specified for the singular noun, such as "a" or "an", will be used when referring to object types (e.g. a recipe). The determiner used when referring to specific instances of object types, such as "Chocolate Chip Cookies", can be specified using the og:determiner meta tag when creating the object.
    2. Plural Noun field is pre-filled with the plural form of the noun corresponding to your initial object type. This will be used when displaying an aggregated story in an Aggregation.
  4. Advanced:
    1. Sample Objects field contains the added sample objects. The first time you walk through wizard for an object type you will have no sample objects. If you are editing an existing object type you will be able to edit your sample object's data. You have the option to debug a sample object. Debugging the object runs it through the ''Object Debugger'' tool. The tool allows you to verify that there are no errors in the page generated to represent your sample object. You can also delete an existing sample object. Add A [OBJECT_NAME] or Add Another [OBJECT_NAME] allows you to add additional sample objects.
    2. Type field allows you to modify the og:type for your object type. It is initially set to YOUR_APP_NAMESPACE:OBJECT_NAME You can modify the object type name portion of the og:type.

Once you have configured your object type the next step is to define an aggregation unit.

Define Custom Properties

Custom properties are additional attributes to further describe a custom object type. Since objects are always visible to the public, do not store any data in an object that is specific to a user, such as a user's private information that should only be available to friends. For such personal data, use Action Properties. Examples of personal data includes properties such as the duration of a run or a user's rating of a restaurant.

Custom properties can also be defined to refer to other objects that have been registered by the same app. For example, a book object may have a property that refers to an author object. Similarly, a recipe object may define a custom property that refers to an ingredient object.

Custom properties of a custom object are static and non-user specific. For example, with the action Run and object Route, the duration of a specific user's run would be an action property while the fixed length of the route would be an object property.

GeoPoint Custom Properties

GeoPoint is a complex type that can be described with latitude, longitude, and altitude. It can be used to specify geospacial information relevant to your objects. For example, a Run can be represented with a single geographic location, or a route defined with an array of GeoPoints.

This information can be used by your application or by Facebook. For example, Facebook uses the GeoPoint object to define the Map Layout Style Aggregation Units that highlight your stories in a map. In addition, Newsfeed and Ticker can also use this type to render more interesting map views.

Note: All objects still need to provide a valid og:image value for situations where maps are not appropriate.

Video Custom Properties

Video has been a large part of the Open Graph ecosystem through stories generated by the built-in "watch" action. However, we understand that there are many cases where custom actions may also have uses for sharing rich video content without needing to define and implement a separate "watch" action. Therefore, we have extended the ability to use the og:video meta tags with your custom open graph objects that will help easily and quickly bring an additional level of rich content to your stories.

In addition to the required Open Graph meta tags for a standard story, the properties below will you customize the right experience for your users:

Property Name Description Required?
og:video Fully qualified URL path to your video content Required
og:video:secure_url Fully qualified Secure URL path to your video content Optional
og:video:height height display value in pixels Optional
og:video:width width display value in pixels Optional
og:video:type list of supported video types Optional

How It Works:

The og:video meta tag is required in order to display a "video play" icon as an overlay to your og:image content. The user behavior will depend on whether the user is browsing Facebook in secure browsing mode (https) or not and how you've configured your video sources and video type.

If you always wish to have a user redirected to the url at og:video or og:video:secure_url, then you should set the og:video:type to "text/html".

If you want the user to be able to play your video inline, on Facebook, then set the appropriate video type like "application/x-shockwave-flash" and make sure you've configure both the og:video and og:video:secure_url tags with the location to the video source. If you are not able to support og:video:secure_url, then users who are in secure browsing mode on Facebook, will be redirected to your og:video location when they attempt to play the video.

The choice of redirecting a user to your site or having the video play inline is up to you and meeting the needs of your users. For example, having a video play inline is a great way to give the user more context as to why they should then click on your story and visit your site.
