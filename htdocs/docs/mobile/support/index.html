Mobile Distribution

Getting Started › Mobile › Mobile Distribution

Introduction

Facebook Platform enables you to get distribution for your mobile apps. The sections below explain the features available across mobile web, iOS and Android.

If you have Facebook Platform app, you will also get distribution there as long as you use the same app id. Apps on Facebook get distribution from Requests, News Feed, Search, and Bookmarks. Websites get distribution via News Feed.

Which Social Channels can I implement?

The table below shows you the features available if you're building a web, iOS or Android app.

Web app iOS native app Android native app
Send a request
Post to News Feed/Wall
Open Graph
Like button

I'm building a web app, where can I get distribution?

Mobile web apps can get distribution from Facebook native apps as well as the mobile web app (facebook.com). The matrix below explains where users will be able to tap through to your web app.

Facebook iPad and iPhone native app facebook.com on iOS* Facebook Android native app facebook.com on Android*
Notifications (via a Request)**
News Feed and Wall
Search
Bookmark

* Tablet distribution via m.facebook.com is coming soon. Tablet traffic is currently sent to www.facebook.com.

** If installed, a push notification via the Facebook native app will also be sent.

I'm building a native iOS app, where can I get distribution?

If you have Single Sign-On integrated, your iOS native app can get distribution from the Facebook native iOS app (iPad and iPhone). The matrix below explains where users will be able to tap through to your native app.

Facebook iPhone and iPad native app
Receiving a request
News feed and Wall
Search
Bookmark

Note that if you have iOS Application Settings filled in, users will be sent to your native app if it shares the same app id as the web app.
