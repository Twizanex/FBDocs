Publishing Permissions

Core Concepts › Authentication › Publishing Permissions

There are two permissions,publish_actions and publish_stream, that allow you to publish stories or activity on behalf of the users of your app. If a user grants you a publishing permission, actions you take on the user’s behalf must be expected by the user and consistent with the user’s actions within your app.

Requesting publish_actions

The publish_actions permission was introduced with the launch of the Open Graph in September, 2011. When a user grants this permission to the app, the app can publish Open Graph stories from that app. Additionally, the app is able to publish the following activity (only available through the Graph API):

  • post to the user's feed
  • upload photo and videos - including creating an album
  • tag photos and videos
  • add likes/comments to stories

If your app requires any publishing abilities not listed above, then you will need to request the publish_stream permission.

Requesting publish_stream

The publish_stream permission is a superset of publish_actions allowing everything that publish_actions allows plus more. Some of the additional publishing capabilities are:

  • posting to a friend's feed
  • posting questions
  • creating notes
  • posting content to events or groups

Since publish_stream is a second screen Auth Dialog permission, make sure you really need the specific capabilities solely offered through publish_stream before requesting it. We have found that requesting permissions that prompt a second Auth Dialog screen can lead to lower conversion rates.

Auth Dialog Implications

The publish_stream permission is considered a second screen permission in the Auth Dialog. This not only creates a less fluid experience for users installing your app, but also allows users to skip the permission. So before requesting the publish_stream permission, please be sure that your app requires its additional publishing capabilities.

In contrast, the publish_actions permission is a first screen permission in the Auth Dialog, which must be accepted in order to install your app. Like all permissions, it can still be revoked by the user in their App Settings.

Calling /<uid>/permissions

If you call the /<uid>/permissions/ Graph API endpoint to see what permissions a user has granted and the user has granted publish_stream, you will also see that they have granted the publish_actions permission. This is due to the fact that publish_stream is a superset of publish_actions.

Permissions in the User App Settings

If you look in your App Settings (Account Settings -> Apps) at an app where you have granted the publish_stream permission, even though it is a superset of publish_actions, only one icon will be displayed for publish_stream. If the app had only requested publish_actions, then you would see a single icon that specifically refers to publish_actions. Both permissions are revokable by the user in their App Settings.

Requesting Both Publishing Permissions

If you attempt to request both the publish_actions and publish_stream permission at the same time, the Auth Dialog will ignore the publish_actions permission and only prompt for the publish_stream permission. This of course will mean that the permission will be prompted in the second Auth Dialog screen and can be skipped by the user.
